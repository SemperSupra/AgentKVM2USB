# Epiphan KVM2USB 3.0: Hardware Architecture & Protocol Report

This report documents the internal hardware components and communication protocols of the Epiphan KVM2USB 3.0, cross-referencing empirical reverse-engineering data with authoritative manufacturer specifications and verified board-level evidence.

## Architecture Overview
The current understood mixed-signal pipeline is:
`VGA (analog) → ISL98002CRZ (ADC) → FPGA (Xilinx Spartan-6 XC6SLX16) → DDR → FX3 (CYUSB3014) → USB`

*Note: The digital (HDMI/DVI) capture path is not yet confirmed and requires further inspection. No specific HDMI receiver IC is assumed at this time.*

## 1. System-on-Chip (SoC) / USB Controller
**Component:** Cypress (Infineon) EZ-USB FX3 (CYUSB3014)  
**Confidence Level:** Confirmed (based on physical inspection and firmware `kvm2usb3.img`)
**Authoritative Source:** [Infineon CYUSB3014 Datasheet](https://www.infineon.com/dgdl/Infineon-EZ-USB_FX3_SuperSpeed_USB_3.0_Peripheral_Controller-DataSheet-v30_00-EN.pdf?fileId=8ac78c8c7d0d8da4017d0ee7be951670)

### Technical Analysis:
The FX3 serves as the primary USB 3.0 interface and system controller. It features an **ARM926EJ-S core** (200 MHz) and **512 KB of on-chip SRAM**.
*   **DMA Engine:** The FX3's General Programmable Interface (GPIF II) provides a high-speed data bus between the FPGA and USB interface, allowing for the high throughput required for uncompressed video capture.
*   **Protocol Stack:** The FX3 likely handles the enumeration of the device as a standard **USB Video Class (UVC 1.1)** and **HID (Human Interface Device)** peripheral.

---

## 2. Video Processing Engine (FPGA)
**Component:** Xilinx Spartan-6 (XC6SLX16)
**Confidence Level:** Confirmed (based on physical inspection)

### Technical Analysis:
The FPGA acts as the bridge between the high-speed video receiver and the USB controller.
*   **Real-time Processing:** The FPGA likely performs color space conversion, scaling, and frame-rate control in hardware logic gates, ensuring the near-zero latency required for KVM operations.
*   **Toolchain Implication:** Reverse-engineering or modifying the FPGA bitstream requires the Xilinx ISE toolchain.

---

## 3. Video Receiver (Digitizer)
**Component:** Intersil ISL98002CRZ (Analog ADC)
**Confidence Level:** Confirmed (based on physical inspection)

### Technical Analysis:
The ISL98002CRZ serves as the analog front-end for video capture.
*   **VGA Capture:** This chip acts as an analog ADC, providing VGA/RGB capture capability.
*   **Digital Capture:** The components for DVI/HDMI (Digital) capture are not yet identified and require further inspection.

---

## 4. Memory
**Component:** ISSI IS43DR16320D (DDR SDRAM)
**Confidence Level:** Confirmed (based on physical inspection)

### Technical Analysis:
*   **Frame Buffering:** The FPGA interfaces with this external DDR SDRAM to buffer frames, managing "tearing" and ensuring a stable video feed even during USB bus congestion.

---

## 5. Analog Section
**Component:** Shielded analog front-end section
**Confidence Level:** Confirmed (based on physical inspection)

### Technical Analysis:
*   **Signal Integrity:** A physically shielded region is present on the board to protect the analog signal integrity from electromagnetic interference generated by the high-speed digital components.

## 6. Communication Protocol Stack
**Confidence Level:** Inferred (based on empirical evidence)
**Empirical Evidence:** USB Descriptor analysis via `dump_usb2.py` and HID enumeration via `hidapi`.

### Protocol Mapping:
| Interface | Endpoint | Protocol | Logic |
| :--- | :--- | :--- | :--- |
| **Video** | Interface 1/2 | **UVC** | Standard Isochronous transfers for video stream. |
| **Keyboard** | Usage 0x101 | **HID** | Custom 8-byte report format on Vendor-Defined Interface. |
| **Mouse** | Usage 0x102 | **HID** | Standard 4-byte Relative Mouse report. |
| **System** | Usage 0x104 | **Vendor** | Uses `GET_FEATURE_REPORT` for hardware diagnostics (resolution, LEDs). |

## 7. Implications for Reverse Engineering
*   **FPGA Toolchain:** The presence of a Xilinx Spartan-6 means the older Xilinx ISE toolchain (not Intel/Altera) is required for any bitstream analysis or modification.
*   **USB Interface:** The Cypress FX3 is likely the primary USB interface.
*   **Protocol Implementation:** Standard UVC and HID protocols are likely implemented directly via the FX3 firmware.
*   **Bitstream:** The FPGA bitstream (`kvm2usb3.bin`) is likely proprietary and specific to the Spartan-6 architecture.

---

## 8. Conclusion
The Epiphan KVM2USB 3.0 is built on a mixed-signal architecture. Analog VGA capture is handled by an ISL98002CRZ ADC, which feeds into a Xilinx Spartan-6 FPGA for bridging and video processing, buffering via external DDR SDRAM. Dedicated USB 3.0 communication (FX3) manages UVC and HID enumeration. While the analog path is well-understood, the digital video (HDMI) capture path remains unidentified and requires further inspection.
